Tutorial

Lesson 0

Press the plus and minus buttons in the navigation bar (next to the Tutorial button) and see what it means for the number of columns on the screen.

(If using a very thin screen, click the minus button while on this tutorial page to get only one column on the screen.)

Try to open some columns by clicking on the various links in the columns.

Try to open more columns than can fit inside the screen at once and then cycle through them using either the left and right arrow keys or by clicking the margins of the page (with the arrow symbols in them).

Lesson 1

First create an account. (You can use a fake e-mail address to make a temporary account for testing purposes.) Click on 'Sign Up' and fill out the form.

Once this is done, open this website in a new tab. Note that you can switch easily between tabs using Ctrl+Tab, especially if you have only two tabs open.

View the initial column with the title "Entities." This is the fundamental entity in the database which represents the category of all entities (including itself). It is the "category of all things" in other words.

Note that the 'Full title' says: "Category ▸ Entities." This means that the entity in question is of the type "Category."

In the Subcategories tab, select the category called "Music" (by clicking on that word).

Note that its 'Full title' says: "Category ▸ Music," This entity thus represents the category of things related to music.

In the Subcategories tab of the "Music" column, select "Rock" (by clicking on that word).

Note that its 'Full title' says: "Category ▸ Rock (Music)." The parentheses are thus used here to disambiguate what "Rock" refers to. It refers to the genre of music, not to stone, nor to a certain kind of motion, nor to a tool used for spinning.

Lesson 2

Resume or repeat lesson 1 such that you have opened the "Category ▸ Rock (Music)" entity.

Note that under its ID, it says: "Template: {<Title>} (<Supercategory>)."
